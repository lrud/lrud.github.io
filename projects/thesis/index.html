Develop and evaluate LSTM models for Bitcoin DVOL forecasting, benchmarked against 13 linear and tree-based models. Validate through statistical metrics (R², RMSE, MAPE, directional accuracy) and economic significance (VaR backtesting).

Abstract

This master’s thesis presents a comprehensive 17-model benchmark framework for Bitcoin implied volatility (DVOL) forecasting. The research combines traditional financial econometrics with modern deep learning, developing LSTM neural networks that achieve 92.87% R² (MAPE 6.33%, RMSE 5.05, 62% directional accuracy) on out-of-sample test data. A key discovery documents a 13.3% R² degradation artifact in PyTorch DataParallel multi-GPU evaluation, the first such documentation in financial machine learning literature.

Critical Discoveries

  1. DataParallel Evaluation Artifact – Identified 13.3% R² degradation in multi-GPU LSTM evaluation. Corrected evaluation reveals LSTM competitiveness vs. traditional methods.

  2. Jump Features Are Redundant – Contrary to initial hypothesis, explicit Lee-Mykland (2008) jump detection provides no predictive value. Lagged volatility features alone achieve optimal performance.

  3. VaR Backtesting Validation – Out-of-sample risk management validation:

    • 95% VaR: 3.31 (5.01% exceedance, Kupiec test PASS)
    • 99% VaR: 5.66 (1.01% exceedance, Kupiec test PASS)

Tree-Based Models (8 specifications):

  • Random Forest (lags + jumps): 0.9492 R² (best overall)
  • Random Forest (lags): 0.9485 R²
  • XGBoost (lags): 0.9429 R²

LSTM Models (6 specifications):

  • LSTM market_lags (512×7): 0.9287 R² (best LSTM)
  • LSTM jump_aware (512×7): 0.7986 R²
  • LSTM market_jumps (512×7): 0.6100 R²

Key Insight: Volatility persistence (lagged features) is the dominant predictive signal across all model classes.

Dataset & Methodology

Data

  • 41,344 hourly samples (June 2021–present)
  • Source: Deribit API (BTC options volatility surface)
  • Target: Next-day DVOL (Deribit 30-day implied volatility index)
  • Features: Lagged DVOL, realized volatility, on-chain metrics, volatility spreads

Model Architecture (Best Performing)

class LSTMVolatility(nn.Module):
    def __init__(self, input_dim=7, hidden_dim=512, num_layers=7):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size=input_dim,
            hidden_size=hidden_dim,
            num_layers=num_layers,
            batch_first=True,
            dropout=0.2
        )
        self.fc = nn.Linear(hidden_dim, 1)

Input: (batch_size, sequence_length=720h, features=7)
Output: (batch_size, 1) – Normalized DVOL forecast (24h ahead)

Training Pipeline

  • Framework: PyTorch 2.0+
  • Normalization: 72-hour rolling window (optimal for regime adaptation)
  • Data Split: Time-series split (80% train, 20% test, no shuffling)
  • Hardware: NVIDIA A100 with multi-GPU DataParallel

Academic Contributions

  1. Unified Benchmark Framework – First systematic comparison of 17 models (linear, tree, LSTM) on identical Bitcoin volatility dataset with consistent preprocessing.

  2. DataParallel Artifact Documentation – First report of 13.3% R² degradation in PyTorch multi-GPU evaluation for financial time series models.

  3. VaR Backtesting Pipeline – Academic-standard risk management validation with Kupiec test for exceedance rate validation.

  4. Jump Detection Analysis – Rigorous Lee-Mykland (2008) implementation demonstrating jump features provide no predictive value for Bitcoin volatility.
