Project Overview

weather_poly is an open-source research repository implementing a comprehensive 17-model benchmark framework for Bitcoin implied volatility (DVOL) forecasting. The project provides a unified pipeline for training, evaluating, and comparing linear models (HAR-RV, OLS), tree-based methods (Random Forest, XGBoost), and deep learning approaches (LSTM).

Key Contributions

1. Unified Benchmark Framework

  • 17 models trained and evaluated on identical preprocessing pipeline
  • 5 Linear Models: HAR-RV (canonical), OLS variants with different feature sets
  • 8 Tree Models: Random Forest and XGBoost with lag/jump configurations
  • 6 LSTM Models: Market lags, jump-aware, and hybrid configurations

2. DataParallel Artifact Discovery

Identified and documented a 13.3% R² degradation in PyTorch multi-GPU LSTM evaluation. This is the first documentation of this artifact in financial machine learning literature. The corrected evaluation shows LSTM competitiveness vs. traditional methods.

3. VaR Validation Pipeline

Out-of-sample risk management validation framework:

  • 95% VaR: 3.31 (5.01% exceedance, Kupiec test PASS)
  • 99% VaR: 5.66 (1.01% exceedance, Kupiec test PASS)

4. Multi-Window Normalization Analysis

Systematically tested rolling windows (72h, 168h, 336h, 720h, 1440h). 72-hour (3-day) window proved optimal for R² prediction, balancing regime adaptation with feature-target relationship preservation.

Getting Started

# Train best model (market_lags, 512×7)
python src/main.py --config configs/market_lags_512x7.yaml

# Evaluate trained model
python src/evaluate.py --model models/final/market_lags_512x7.pth

# Run full benchmark suite (17 models)
python scripts/benchmarking/run_all_benchmarks.py
